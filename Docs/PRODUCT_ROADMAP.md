# WCS Apple ecosystem roadmap

## M1 — Shared foundation + Memory Atlas shell

Outcome: buildable app foundation.

- Create `WCSCore` package.
- Add shared design system, domain models, repositories, API client, media upload, CloudKit boundary.
- Create Memory Atlas app shell with dashboard, profile flow, artifact list, session builder/player placeholders.

## M2 — Memory profile + artifact CRUD

Outcome: real user data flow.

- Wire profile list, profile detail, artifact creation.
- Add API-backed repositories.
- Add signed media upload flow.

## M3 — Session builder + player

Outcome: core therapeutic experience.

- Build guided reminiscence session sequence builder.
- Add active session timer, prompts, mood before/after, notes.

## M4 — Widgets, Live Activities, notifications

Outcome: Apple-native engagement loop.

- Add WidgetKit “Today’s Memory”.
- Add ActivityKit active guided session state.
- Add caregiver reminders.

## M5 — CloudKit backup + offline cache

Outcome: resilience and sync.

- Mirror profile summaries and artifact metadata into CloudKit.
- Add local cache and sync coordinator.

## M6 — visionOS spatial prototype

Outcome: premium differentiation.

- Add Vision Pro target.
- Add RealityKit memory room boundary.
- Add spatial scenes and guided immersive memory mode.

## M7 — ScholarSphere Studio

Outcome: WCS content platform.

- Project dashboard, lesson editor, asset manager, assessment composer, publishing service boundary.

## M8 — Presence Play

Outcome: shared spatial therapy/learning prototype.

- Session lobby, narrative engine, scene orchestration, reflection summary, single-user immersive prototype first.
